# الجزء الرابع - الدرس الاول: ما هي إدارة الإصدارات؟

## تعريف
عملية البرمجة هي عملية تفاعلية تحتمل الكثير من التجريب على الكود والتحسين المستمر خلال شيء يسمى إصدارات. فمثلا لو كان لديك برنامج على الهاتف أو على الجهاز فإن ذلك البرنامج يحمل معه رقم النسخة version.

في معظم الشركات البرمجية يعتمدون على شيء يسمى إدارة الإصدارات بحيث يستطيعوا من خلاله مواكبة تطور برنامج معين وتتبع إصداراته المختلفة، وأيضا تستخدم إدارة الإصدارات كنسخ احتياطية في حالة فقدان شيء من الكود المعمول عليه على أجهزة المبرمجين.

هناك الكثير من البرامج التي تستطيع من خلالها إدارة إصدارات المشروع الذي تعمل عليه بعضها "مركزي" أي يجب أن يتواجد سيرفر واحد يتصل به كافة المبرمجين ومن خلاله يتم إدارة الإصدارات، ومنها "اللامركزي" أي تستطيع أن تعمل من عدة أجهزة ولاحقا تستطيع هذه الأجهزة أن تتواصل مع بعض بشكل فردي وأيضا تتيح لك اللامركزية أن تعمل وأنت مفصول عن الإنترنت وتقوم بتحديث عملك لاحقا عندما تقوم بالاتصال بالأنترنت، في هذه الدورة سنأخذ بالتفصيل أحد هذه البرامج وأشهرها ألا وهو جت git وهو بالطبع "لا مركزي".

عندما كنت تبرمج قبل أن تتعرف على جت فقد كنت تعمل كل عملك على فوادر واحد وهو فولدر البرنامج فذلك الفولدر هو كل ما لديك إذا عدلت عليه وليس لك نسخة احتياطية فلن تستطيع أن تعود لما كان عليه سابقا، والأسوأ من ذلك إذا تم إلغاءه فسيضيع كل عملك. من خلال جت ستقوم بالعمل على ثلاث فولدرات للكود، أولها الفولدر الذي تحدثنا عنه وهو الفولدر الذي يحمل ملفات البرنامج ويسميه جت Working Directory، وجت تتيح لك فولدرين أخرين (مجازاً سأسميهم فولدرات لسهولة التعامل معهم وفهمهم) الأول هو Staging Index وهو مرحلة متوسطة بين فولدر المشروع الحالي والفولدر الثالث ألا وهو Repository أو اختصارا Repo وهو المستودع الذي سيحمل كل إصدارات البرنامج الذي عملت عليها سابقا بما فيها أخر إصدار.

لاحظ أن working directory و staging index or area سيحملون نسخة واحدة من البرنامج في الوقت الواحد أما ال Repo فسيحمل كل النسخ التي ترسلها له ويقوم بترتيبها لك لتعود إليها متى شئت. سنخوض في ذلك لاحقاً.

## إعداد جت git  على جهازك:
قد تكون من خلال الدروس الأولى في هذا الكورس قد قمت بتنصيب برنامج git bash على جهازك، إذا لم تكن فعلت ذلك فتستطيع أن تبدأ الأن بهذا التنصيب قبل متابعة الإعدادات:

للويندوز: 
https://youtu.be/xvmn3pw4WQ8


للماك:
https://youtu.be/fioRoXuZhRM


حسنا الأن بعد آن أتممت تنصيب ال git  دعنا نقوم ببعص الإعدادات: 
يجب أن تقوم بإعطاء جت فكرة عنك من ناحية الاسم والإيميل، لنرى ذلك:

إبدا جت في ويندوز أو على ماك فقط إبدأ ال  Terminal.

الأن يريد جت أن يتعرف عليك، أعلمه عن اسمك "سترى ذلك الاسم لاحقا من خلال العمل على جت وسيكون مفيد عندما تعمل مع أكثر من مبرمج على مشروع واحد" اطبع اسمك بين علامات التنصيص ونفذ الأمر:

    git config --global user.name "<Your-Full-Name>"

أعلمه عن بريدك الإلكتروني :"سترى ذلك أيضا لاحقا من خلال العمل على جت وسيكون مفيد عندما تعمل مع أكثر من مبرمج على مشروع واحد" اطبع إيميلك بين علامات التنصيص ونفذ الأمر: 

    git config --global user.email "<your-email-address>"

هنا يتم تحضير جت ليريك النتائج بالألوان:

    git config --global color.ui auto

وهنا تحدد كيف سيتم عرض الاختلاف بين الإصدارات المختلفة (سنفهم ذلك لاحقا) :

    git config --global merge.conflictstyle diff3


والأن لتتأكد من الإعدادات تمت بنجاح نفذ الأمر التالي:

    git config --list


كل الإعدادات السابقة اختيارية ولكن ضرورية جدا لتحصل على بيئة عمل صحيحة ونظيفة.

بقي أن نقوم بأخر إعداد لنا ألا وهو تجهيز ال git bash الذي تم تنصيبه على ويندوز أو تجهيز ال Terminal على ماك ليعطينا معلومات مفيدة عن تحركاتنا بين فروع الكود (سنفهم ذلك لاحقا):

قم يتنزيل الملف التالي:
http://video.udacity-data.com.s3.amazonaws.com/topher/2017/March/58d31ce3_ud123-udacity-terminal-config/ud123-udacity-terminal-config.zip


للويندوز تابع الفيديو التالي:
https://youtu.be/AtiNxmmh5TE


للماك تابع تابع الفيديو من:
https://youtu.be/UDFxhy1gF34

وأخيرا سنقوم بتحديد محرر ل جت، سيقوم الجت باستخدام هذا المحرر إذا احتاج أن يحصل منك على معلومات إضافية فأسهل لك أن تعرّف محرر خاص بك: (فقط استخدم واحداً مما يلي وليس كلها):

إبدأ git bash أو terminal:

. لتجهيز atom على أنه المحرر الافتراضي:

    git config --global core.editor "atom --wait"

. لتجهيز sublime على أنه المحرر الافتراضي على ويندوز:

    git config --global core.editor "'C:/Program Files/Sublime Text 2/sublime_text.exe' -n -w"

. لتجهيز sublime على أنه المحرر الافتراضي على ماك:

    git config --global core.editor "subl -n -w"

. لتجهيز Visual Studio Code على أنه المحرر الافتراضي:

    git config --global core.editor "code --wait"


 بذلك أصبح كل شيء لديك جاهزا لتبدأ مع جت :smiley:
